[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let me walk you through our system architecture. First, any user query goes through the Security Checkpoint node to filter out threats and redact PII. If it is safe, the prompt reaches the Care Coordinator orchestrator agent. The coordinator then delegates specialized medical tasks to the medication agent or appointment agent. These agents interface with our local database via custom MCP server tools. When a modification is requested, the Post-Coordinator Check identifies the pending action and routes the path to the caregiver approval HITL node, prompting a human caregiver to approve or reject the action.

Meanwhile, our MCP server tools handle local database persistence with complete isolation, representing a production-grade healthcare architecture.
